# Production-Backend-Go

A production ready backend system written in go with many backend concepts such as MVC architecture, concurrency, producer consumer architecture, logging, monitoring, caching etc.

Building using Echo (framework), pgx (postgreSQL driver) , zerologg, validator, koanf (config management library), testify (for unit tests), test containers (abstracts docker for testing dbs)

A boilerplate setup to make go projects faster.

A mono repo setup because we use type safe stuff and keep everything in 1 repo

refer go standard layout for more stuffs
